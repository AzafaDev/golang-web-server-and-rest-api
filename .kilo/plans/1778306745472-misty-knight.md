# Rencana Implementasi Blog API — Go Backend

## Tujuan
Membangun REST API blog sederhana dengan Go, menggunakan PostgreSQL (Neon) sebagai database, chi sebagai router, dan pgx v5 sebagai driver database. Fokus pada best practices: separation of concerns, connection pooling, parameterized query, middleware, dan graceful shutdown.

## Status Saat Ini
- `backend/go.mod` sudah memiliki dependensi: `chi/v5`, `pgx/v5`, `godotenv`
- `backend/.env` sudah berisi `DATABASE_URL` untuk Neon PostgreSQL
- `backend/main.go` kosong (hanya `func main() {}`)
- File `models.go`, `store.go`, `handlers.go` belum ada

## Rencana Implementasi

### 1. Buat `backend/models.go` — Definisi Data
- Struct `Post` dengan field: `ID`, `Title`, `Content`, `CreatedAt`, `UpdatedAt` + struct tag `json:"..."`
- Struct `CreatePostInput`: `Title`, `Content` (string biasa)
- Struct `UpdatePostInput`: `Title`, `Content` (pointer `*string` untuk partial update/PATCH)
- Struct `APIError` untuk respons error standar `{"error": "message"}`

### 2. Buat `backend/store.go` — Layer Database
- Struct `Store` menyimpan `*pgxpool.Pool`
- Fungsi `NewStore(ctx)`:
  - Baca `DATABASE_URL` dari environment variable
  - Parse config, set `MaxConns=10`, `MinConns=2`
  - Buat pool, ping database
  - Auto-create tabel `posts` jika belum ada (`CREATE TABLE IF NOT EXISTS`)
- Method CRUD:
  - `CreatePost(ctx, input)` — parameterized query dengan `RETURNING`
  - `GetPosts(ctx)` — iterasi `rows`, `defer rows.Close()`, cek `rows.Err()`
  - `GetPostByID(ctx, id)` — handle `pgx.ErrNoRows` untuk 404
  - `UpdatePost(ctx, id, input)` — fetch dulu, timpa field non-nil, `UPDATE ... RETURNING`
  - `DeletePost(ctx, id)` — cek `RowsAffected() == 0`
- Method `Close()` untuk menutup pool

### 3. Buat `backend/handlers.go` — HTTP Handler
- Helper: `writeJSON(w, status, data)` dan `writeError(w, status, message)`
- Struct `Handler` dengan field `store *Store` (dependency injection)
- 5 handler method sesuai signature `http.HandlerFunc`:
  - `CreatePost` — decode JSON body, validasi Title tidak kosong, kirim `201 Created`
  - `GetPosts` — kirim semua posts
  - `GetPost` — ambil `{id}` dari `chi.URLParam`, parsing ke int, kirim post atau 404
  - `UpdatePost` — decode JSON body (pointer fields), partial update, kirim hasil
  - `DeletePost` — ambil `{id}`, hapus, kirim `204 No Content` atau 404

### 4. Perbarui `backend/main.go` — Entry Point & Server
- Load `.env` dengan `godotenv.Load()`
- Setup `slog.NewJSONHandler(os.Stdout, nil)` sebagai logger default
- Inisialisasi `NewStore(ctx)`, `defer store.Close()`
- Setup chi router dengan middleware standar:
  - `middleware.RequestID`
  - `middleware.RealIP`
  - `middleware.Logger`
  - `middleware.Recoverer`
  - `middleware.Timeout(60s)`
- Register route di bawah `/api`:
  - `GET /api/posts`
  - `POST /api/posts`
  - `GET /api/posts/{id}`
  - `PUT /api/posts/{id}`
  - `DELETE /api/posts/{id}`
- Konfigurasi `http.Server` dengan `ReadTimeout`, `WriteTimeout`, `IdleTimeout`
- Graceful shutdown: jalankan server di goroutine, tunggu `SIGINT`/`SIGTERM`, shutdown dengan timeout 30 detik

### 5. Verifikasi
- `go build ./...` — pastikan kompilasi berhasil
- `go run .` — jalankan server dan uji endpoint dengan curl

## Keputusan Desain (Why)
| Keputusan | Alasan |
|-----------|--------|
| `*string` untuk UpdatePostInput | Membedakan field tidak dikirim (`nil`) vs dikosongkan (`""`) — standar partial update/PATCH |
| `RETURNING` clause | Satu round-trip database, tidak perlu SELECT ulang |
| `pgxpool` (bukan `database/sql`) | Performance terbaik, connection pool bawaan, tipe asli PostgreSQL |
| Middleware chi standar | Observability (RequestID, Logger), keamanan (RealIP, Recoverer), ketahanan (Timeout) |
| Graceful shutdown | Request yang sedang berjalan selesai sebelum aplikasi mati, cegah data inkonsisten |
| `log/slog` bawaan | Go 1.21+ built-in, format JSON mudah diparse sistem monitoring |
| Tabel auto-create | Memudahkan development/deployment awal (production sebaiknya pakai migrasi) |
